# せんごく タイピング

小学2年生（キーボード触りたて・ローマ字未習）が、楽しくブラインドタッチの素地を身につけるためのブラウザゲーム。

**戦国武将のフルネームをローマ字で打つ**と、その武将が なかま（ずかん）に加わる、集める育成型。`odanobunaga`→織田信長、`takedashingen`→武田信玄 のように、打った名前がそのまま武将になる。武将ごとに**やさしい解説**つき。

## あそびかた

- メニューの「ゆびの たいそう」でホームポジション（asdf / jkl;）に慣れる
- 「せんごくの ことばを あつめる」でステージを選び、出てきた武将名をローマ字で打つ
- 正しく打つと その武将が あらわれて ずかんに登録。ステージを全部あつめると次が解放
- 画面のキーボードは**指ごとに色分け**。次に押すキーが光るので、**ローマ字を知らなくても光に従えば打てる**（手元を見ずに画面で打つ＝ブラインドタッチの素地、小3のローマ字入力の先取り）
- 武将名は漢字に**ふりがな**、**日本語で読み上げ**、**やさしい解説**を表示
- ステージはローマ字の文字数でやさしい順（短いフルネーム→長いフルネーム）に自動で分かれる
- 「おたすけ」で補助を3段階：しょしんしゃ（点滅＋色）→ なれてきた（色のみ）→ たつじん（キーボードをかくす）

## データ

戦国武将 約50名を `src/data/words.ts`（`GameWord`: ローマ字・漢字名・ふりがな・絵文字・解説）に収録。名前と読みは Wikipedia「戦国大名」ほかを参考。所属ステージは `worlds.ts` の文字数範囲から自動算出（手作業の割り当てミスが起きない設計）。武将の追加は配列に1行足すだけ。

## 開発

```bash
npm install
npm run dev     # 開発サーバー（http://localhost:5173/）
npm run test    # ユニットテスト（vitest）
npm run build   # 型チェック＋本番ビルド（dist/）
npm run preview # ビルド成果物をプレビュー
```

ロジック（`src/core`）と画面（`src/ui`）を分離。打鍵判定・指割当・進捗保存・データ不変条件は純粋関数で単体テスト済み。

## デプロイ（GitHub Pages）

`main` に push すると `.github/workflows/deploy.yml` が `dist/` を GitHub Pages へ公開する。
初回のみ GitHub リポジトリの **Settings → Pages → Source: GitHub Actions** を有効化する。
`vite.config.ts` の `base: "./"` によりサブパス配信でもそのまま動く。
